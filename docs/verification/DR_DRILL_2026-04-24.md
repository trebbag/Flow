## Scope

Point-in-time recovery drill for staging Azure Database for PostgreSQL Flexible Server using a disposable restore target.

- Source server: `flow-staging-pg`
- Resource group: `flow-staging-rg`
- Restore target: `flow-staging-pitr-20260424180449`

- Test marker: `codex-pitr-20260424180449-1880`

No database passwords or connection strings are recorded in this evidence file.

## Procedure

1. Recorded a restore point timestamp before the test mutation.
2. Inserted a single marker row into the source `EventOutbox` table after that restore point.
3. Created a disposable Azure Flexible Server restore target from `flow-staging-pg` at the restore point.
4. Added a temporary client firewall rule to the disposable target because restored firewall rules were empty.
5. Connected to the restored server and queried for the marker row.
6. Deleted the marker row from the source staging database.
7. Deleted the disposable restore target.
